# Plan 01-02: Compliance Rule Engine + Gemini Remediation

<objective>
Build the deterministic compliance evaluation engine with configurable rule tables in PostgreSQL, implement the generic rule evaluator, create the Gemini-powered remediation text generator, and expose compliance check endpoints. The compliance engine runs automatically after extraction — no LLM is used for any compliance decisions.
</objective>

<must_haves>
- Compliance rules stored as configurable rows in PostgreSQL (not hardcoded)
- Generic evaluation engine that reads rules and evaluates shipment data
- MVP rules seeded for EU (Netherlands) and US corridors
- Traffic-light status: per-shipment rollup + expandable per-check breakdown
- Gemini-generated human-readable remediation for failing checks
- Compliance API endpoint returning structured results
</must_haves>

<action>
**backend/app/schemas/compliance.py:**
Create Pydantic response models:

```python
class CheckResult(BaseModel):
    rule_id: int
    rule_name: str
    check_type: str
    severity: str          # 'red', 'amber', 'green'
    passed: bool
    message: str           # Short status message
    remediation: Optional[str] = None  # Gemini-generated guidance (only if failed)
    ingredient_name: Optional[str] = None  # Which ingredient triggered the failure

class ComplianceReport(BaseModel):
    shipment_id: str
    destination_country: str
    overall_status: str    # 'red' if any red fails, 'amber' if any amber fails, else 'green'
    checks: list[CheckResult]
    checked_at: str        # ISO timestamp
    summary: str           # "3/7 checks passed" style summary
```

**backend/app/services/compliance.py:**
Create `ComplianceService` class:

1. `async def evaluate_shipment(self, shipment_id: str, destination_country: str, db: AsyncSession) -> ComplianceReport`:
   - Fetch shipment and its extracted_data from DB
   - Fetch all active rules for `destination_country`
   - For each rule: call `_evaluate_rule(rule, shipment_data, ingredients)`
   - Compute `overall_status`: RED if any red check fails, AMBER if any amber fails, else GREEN
   - For failed checks: call `RemediationService.generate(rule, shipment_context)`
   - Return ComplianceReport

2. `def _evaluate_rule(self, rule: ComplianceRule, shipment_data: dict, ingredients: list) -> bool`:
   - Implement generic condition evaluation:
     - `equals`: `data[field] == value`
     - `not_null`: `data[field] is not None`
     - `is_null`: `data[field] is None`
     - `in_list`: `data[field] in json.loads(value)`
     - `not_in_list`: `data[field] not in json.loads(value)`
   - For ingredient-level rules (like scheduling): iterate all ingredients and check each
   - Return True if the condition passes (compliant), False if it fails

3. `def _compute_overall_status(self, results: list[CheckResult]) -> str`:
   - If any result with severity='red' has `passed=False`: return 'red'
   - If any result with severity='amber' has `passed=False`: return 'amber'
   - Else: return 'green'
</action>

<action>
**backend/app/services/remediation.py:**
Create `RemediationService` class:

1. `__init__(self, api_key: str, model_name: str)` — initialize google-genai client
2. `async def generate(self, rule: ComplianceRule, shipment_context: dict) -> str`:
   - Build prompt from rule's `remediation_template` + shipment context:
     ```
     You are a pharmaceutical regulatory compliance advisor. A compliance check has failed for a pharmaceutical export shipment.
     
     Failed Rule: {rule.rule_name}
     Rule Description: {rule.description}
     Destination: {shipment_context['destination_country']}
     Product: {shipment_context['product_name']}
     Template: {rule.remediation_template}
     
     Generate a clear, actionable 2-3 sentence explanation for a pharma export manager explaining:
     1. WHY this check failed
     2. WHAT specifically needs to be done to fix it
     3. Any relevant regulatory reference
     
     Be specific and practical. Do not use generic advice.
     ```
   - Call Gemini with `response_mime_type="text/plain"`
   - Return the generated remediation text
   - If API fails: fall back to the `remediation_template` itself (still useful)

**Integration point:** ComplianceService calls RemediationService for each failed check.
</action>

<action>
**backend/app/api/compliance.py:**
Create compliance router with:

`GET /api/shipments/{shipment_id}/compliance`:
- Accept query param `destination` (country code, default from shipment)
- Check shipment exists and has extracted_data (status >= 'extracted')
- If not extracted yet: return 400 "Extraction must complete before compliance check"
- Call `ComplianceService.evaluate_shipment(shipment_id, destination, db)`
- Return `ComplianceReport` with overall_status, all check results, and remediation text for failures
- Store compliance results in shipment record for caching

`POST /api/shipments/{shipment_id}/compliance/recheck`:
- Force re-evaluation (useful after user corrects extraction data via HITL)
- Same logic as GET but clears cached results first

Register compliance router in `main.py`.
</action>
