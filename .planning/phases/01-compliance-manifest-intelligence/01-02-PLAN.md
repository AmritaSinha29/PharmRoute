# Plan 01-02: Compliance Engine & Remediation (Elixir)

<objective>
Build the deterministic compliance evaluation engine using Ecto for configurable rule storage, and implement the generic rule evaluator in Elixir. Integrate Gemini to generate human-readable remediation text for failed checks.
</objective>

<must_haves>
- Ecto Schema `ComplianceRule` backing a PostgreSQL table.
- Seeded MVP rules for EU (NLD) and US (USA) corridors.
- `PharmRoute.Compliance` Elixir context containing evaluation logic.
- Gemini integration used ONLY for generating remediation advice (no LLM decision making).
</must_haves>

## Tasks

### Task 1: Compliance Rule Ecto Schema

<read_first>
- .planning/phases/01-compliance-manifest-intelligence/01-RESEARCH.md
</read_first>

<action>
Generate a migration for rules: `mix ecto.gen.migration create_compliance_rules`.
Table structure:
- `country_code` (string, size: 3)
- `check_type` (string)
- `rule_name` (string)
- `description` (text)
- `condition_field` (string)
- `condition_operator` (string)
- `condition_value` (text/string)
- `severity` (string)
- `remediation_template` (text)
- `is_active` (boolean, default: true)

Create `lib/pharm_route/compliance/rule.ex` schema matching the migration.
</action>

<acceptance_criteria>
- Migration exists and uses appropriate data types.
- Schema module `PharmRoute.Compliance.Rule` uses `Ecto.Schema`.
</acceptance_criteria>

### Task 3: Elixir Compliance Context

<read_first>
- lib/pharm_route/compliance/rule.ex
</read_first>

<action>
Create `lib/pharm_route/compliance.ex`.
Implement `evaluate_shipment(shipment_data, destination_country)`:
1. Fetch active rules for the destination.
2. Evaluate each rule deterministically (e.g., using `String.equivalent?` or matching logic based on `condition_operator`).
3. Return a Map representing the compliance report: overall status (red/amber/green) and a list of check results.

If a check fails, call `PharmRoute.AI.Gemini.generate_remediation(rule, shipment_data)` to get human-readable instructions. Ensure you create this new function in the Gemini module.
</action>

<acceptance_criteria>
- `PharmRoute.Compliance.evaluate_shipment/2` exists and returns a structured map or struct.
- Deterministic evaluation does not query the LLM to determine pass/fail, only for the text generation after a fail.
</acceptance_criteria>
