# Plan 01-01: FastAPI Backend + Gemini Extraction Pipeline

<objective>
Set up the FastAPI backend with PostgreSQL, implement the Gemini-powered BMR extraction pipeline with Pydantic structured output, build the substance lookup database with seed data, and implement the async job + SSE pattern for real-time progress updates. This plan delivers the full backend infrastructure plus the extraction → validation → HITL workflow.
</objective>

<must_haves>
- FastAPI app running with PostgreSQL connection
- Gemini PDF extraction producing structured JSON with confidence scores
- Substance lookup table seeded with demo substances + top 50 APIs
- Async background task pipeline: upload → extract → validate → store
- SSE streaming endpoint for real-time job progress
- HITL confirmation endpoint (PUT /shipments/{id}/confirm)
- Packing list export endpoint (POST /shipments/{id}/export) — only works after HITL confirmation
- Zero training-data retention: raw PDFs deleted after extraction
</must_haves>

<action>
**backend/app/core/background.py:**
Create job queue manager:
```python
import asyncio
from typing import Dict

job_queues: Dict[str, asyncio.Queue] = {}

async def create_job_queue(job_id: str) -> asyncio.Queue:
    queue = asyncio.Queue()
    job_queues[job_id] = queue
    return queue

async def push_status(job_id: str, step: str, progress: int, message: str, **extra):
    if job_id in job_queues:
        await job_queues[job_id].put({
            "step": step, "progress": progress, "message": message, **extra
        })

async def cleanup_queue(job_id: str):
    job_queues.pop(job_id, None)
```

**backend/app/api/upload.py:**
Create upload router with:

`POST /api/upload`:
- Accept `UploadFile` (PDF only, max 50MB)
- Validate file type (application/pdf)
- Create `Shipment` record (status='pending')
- Create `Job` record (type='extraction', status='pending')
- Start `BackgroundTasks.add_task(run_extraction_pipeline, job_id, shipment_id, file_content)`
- Return `{"job_id": job_id, "shipment_id": shipment_id}`

`run_extraction_pipeline(job_id, shipment_id, file_content)`:
- Push status: "uploading", 100, "File received"
- Push status: "extracting", 10, "Parsing batch manufacturing record..."
- Call `ExtractionService.extract_from_pdf(file_content)`
- Push status: "extracting", 60, f"Extracted {len(ingredients)} ingredients"
- Push status: "validating", 70, "Checking substance database..."
- Call `SubstanceLookupService.validate_ingredients(ingredients)`
- Push status: "validating", 90, "Validation complete"
- Update Shipment with extracted_data, status='extracted'
- Update Job with status='complete', result=extraction_data
- Push status: "complete", 100, "Extraction complete", result_id=shipment_id
- Delete raw PDF from memory (do NOT persist to disk)
- Cleanup queue

**backend/app/api/jobs.py:**
Create jobs router with:

`GET /api/jobs/{job_id}/stream`:
- Return `StreamingResponse` with `media_type="text/event-stream"`
- Stream from `job_queues[job_id]`
- Format each message as `data: {json}\n\n`
- Close stream on "complete" or "error" status
- If job_id not found: check DB for completed job and return final status

**backend/app/api/shipments.py:**
Create shipments router with:

`GET /api/shipments/{id}`:
- Return shipment with extracted_data

`PUT /api/shipments/{id}/confirm`:
- Set `extraction_confirmed=True`, `confirmed_at=now()`, `status='confirmed'`
- Return updated shipment

`POST /api/shipments/{id}/export`:
- Check `extraction_confirmed is True` — if not, return 400 "HITL confirmation required before export"
- Generate packing list JSON from confirmed extracted_data + substance regulatory identifiers
- Set `packing_list_generated=True`, `status='exported'`
- Return packing list data

**backend/app/core/security.py:**
- Middleware to add `Cache-Control: no-store` header on all responses
- Log sanitization: strip `extracted_data` from any log output
- File validation: reject non-PDF uploads, enforce size limit
</action>
