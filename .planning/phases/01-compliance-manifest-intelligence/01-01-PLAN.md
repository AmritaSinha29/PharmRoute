# Plan 01-01: Phoenix Setup, Ecto & Gemini API Extraction

<objective>
Initialize the Phoenix web application, set up the database using Ecto, model the `Substance` and `Shipment` schemas, and implement the Gemini 1.5 Pro integration module for structured BMR extraction in Elixir.
</objective>

<must_haves>
- Phoenix 1.7 project generated successfully.
- Ecto schemas ready and migrations run for `Substances` and `Shipments`.
- Seed file (`seeds.exs`) containing Amoxicillin/Clavulanate and 10+ pharma APIs.
- Elixir module `PharmRoute.AI.Gemini` capable of submitting a PDF (Base64) to the REST API and receiving parsed JSON ingredients.
</must_haves>

### Task 4: Gemini HTTP Client in Elixir

<read_first>
- lib/pharm_route/manifest.ex
</read_first>

<action>
Create `lib/pharm_route/ai/gemini.ex`.
Provide a function `extract_bmr(base64_pdf_content)`:
- Reads system environment variable `GEMINI_API_KEY`.
- Uses `Req.post!` to send a request to `https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-pro:generateContent?key=...`.
- Payload structure must enforce JSON scheme using `response_mime_type: "application/json"`.
- Asks LLM to extract product name, DOSAGE, batch date, and a list of ingredients (name, cas_number, role, confidence).
- Returns `{:ok, parsed_map}` on success.
</action>

<acceptance_criteria>
- `lib/pharm_route/ai/gemini.ex` contains `Req.post!` and mentions `generativelanguage.googleapis.com`.
- Contains specific JSON payload instructions for Gemini model.
</acceptance_criteria>
