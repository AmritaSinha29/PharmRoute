# Plan 01-03: Phoenix LiveView UI

<objective>
Build the Real-Time UI using Phoenix LiveView. Implement drag-and-drop BMR upload, real-time extraction progress via PubSub broadcast, the HITL confirmation table, and the final Compliance traffic-light dashboard.
</objective>

<must_haves>
- LiveView handling drag-and-drop file upload (`allow_upload` and `<.live_file_input>`).
- Real-time UI updates fueled by `Phoenix.PubSub` messages sent from an Elixir `Task`.
- HITL confirmation UI.
- Traffic-light visualization using Tailwind CSS classes.
</must_haves>

## Tasks

### Task 1: UI Router & Core Components

<read_first>
- lib/pharm_route_web/router.ex
- lib/pharm_route_web/components/core_components.ex
</read_first>

<action>
In `lib/pharm_route_web/router.ex`, add routes:
- `live "/", UploadLive`
- `live "/dashboard/:shipment_id", DashboardLive`

In `lib/pharm_route_web/components/core_components.ex`, ensure Tailwind-based UI components (buttons, tables, modals) exist or add simple helpers for traffic light indicators (e.g., `<.traffic_light status={@status} />`).
</action>

<acceptance_criteria>
- `router.ex` contains routes mapping to `UploadLive` and `DashboardLive`.
</acceptance_criteria>

### Task 2: UploadLive & Background Processing (OTP)

<read_first>
- lib/pharm_route/manifest.ex
- lib/pharm_route_web/live/upload_live.ex
</read_first>

<action>
Create `lib/pharm_route_web/live/upload_live.ex`.
1. Employs `allow_upload(:bmr, accept: ~w(.pdf), max_entries: 1)`.
2. UI implements a dropzone via native LiveView uploads.
3. On form submit:
   - Create a `Task.async` (or `Task.start`) that hooks into `PharmRoute.Manifest` to extract the PDF text using the Gemini module from Plan 1.
   - Use `Phoenix.PubSub.broadcast/3` inside the task to send `{:progress, "Extracting...", 30}` updates back to the LiveView.
4. The `UploadLive` module uses `handle_info` to listen for these PubSub updates and mutates the `@progress` socket assign, automatically rendering a progress bar to the user without polling.
5. On completion, redirect to `/dashboard/:shipment_id`.
</action>

<acceptance_criteria>
- `UploadLive` executes `allow_upload` for PDFs.
- Elixir `Task` is spawned for processing to prevent blocking the socket.
- `handle_info` captures and assigns progress state.
</acceptance_criteria>

### Task 3: DashboardLive (HITL & Compliance)

<read_first>
- lib/pharm_route_web/live/dashboard_live.ex
</read_first>

<action>
Create `lib/pharm_route_web/live/dashboard_live.ex`.
1. Loads the `Shipment` by ID.
2. If `extraction_confirmed` is false, show the HITL UI: an HTML table of extracted substances. Include a "Confirm & Analyze Compliance" button.
3. On confirm, update the shipment and run `PharmRoute.Compliance.evaluate_shipment`.
4. If confirmed, render the Compliance Traffic-light UI showing overall status and a list of specific checks (including the Gemini-generated remediation text for any failures).
</action>

<acceptance_criteria>
- Dashboard shows HITL table when unconfirmed.
- Dashboard renders Compliance results appropriately (green/amber/red indicators based on Tailwind) once confirmed.
</acceptance_criteria>
