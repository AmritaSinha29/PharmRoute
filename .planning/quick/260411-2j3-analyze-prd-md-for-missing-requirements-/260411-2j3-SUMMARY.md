# PRD Analysis Report
**Task:** Analyze PRD.md for missing requirements, unclear features, scalability issues, security risks, edge cases.

## 1. Missing Requirements
- **Authentication & Authorization**: The PRD outlines a "Stakeholder Coordination Hub" (P1) but lacks v1 requirements for user roles, access control, and tenant isolation. 
- **Audit Trails**: Pharmaceutical export involves heavy compliance. Changes to automatically generated packing lists or routing overrides *must* have an immutable audit log, which is currently unmentioned.
- **Fail-safes for LLM Extraction**: There's no requirement detailing the UX for when the AI (Gemini 1.5 Pro) cannot confidently extract an API or excipient due to poor PDF or scan quality. A "human-in-the-loop" review queue is missing.

## 2. Unclear Features
- **Simulated Disruption Trigger**: For DEMO-02, it is unclear what the technical trigger is (e.g., an admin dashboard mock button, a scheduled webhook, or hardcoded timing).
- **"Traffic-Light Check" Remediation**: Feature CMP-03 mentions "remediation guidance" for failing checks. It's unclear whether this guidance is pre-authored statically or generated synchronously by the LLM (which risks hallucination).
- **Exact-Match against Generative API**: Under Risks (11), the PRD says substance identification uses exact-match lookup, but section 8.1 claims Gemini 1.5 Pro parses and relies on "multi-step reasoning". The exact handoff mechanism between LLM parsing and strict exact-match lookup is vague.

## 3. Scalability Issues
- **Long-Context Latency**: Passing entire Batch Manufacturing Records (which can be 50-200 pages) into Gemini 1.5 Pro synchronously will lead to high user-facing latency. The UI must decouple this effectively via an async queue instead of blocking the frontend.
- **Vector Search on Compliance**: Pinecone is highly scalable, but regulatory text mapping requires strict logical evaluation (e.g., "if threshold X > 50mg, then rule Y applies"). Embedding search (text-embedding-004) excels at semantic matching but typically fails at strict deterministic rule logic if not backed by a discrete business rule engine.

## 4. Security Risks
- **Data Privacy (BMRs)**: Batch manufacturing records contain highly sensitive proprietary intellectual property. Sending it to external APIs like Google Gemini requires explicit architecture decisions (e.g., utilizing Enterprise Tier with zero-retention agreements) to prevent IP leakage.
- **Role Hijacking**: In a shared hub, an attacker modifying another company's packing list or route could lead to millions in damages. Transport security, encryptions, and strict authorization layers need to be documented.

## 5. Edge Cases Unaddressed
- **Zero Viable Routes**: What does the Route Engine do if a disruption eliminates *all* viable routes that satisfy the strict cold-chain and regulatory timeline (e.g., natural disaster grounding air cargo + port strike)?
- **Mixed Container Cargo**: The PRD assumes a single-product container context. What if the exporter ships Amoxicillin alongside lower-tier standard medical supplies or hazardous chemicals? Compliance checks and cold-chain constraints may heavily conflict across products within the same shipment.

## Recommended Improvements
1. **Add Human-in-the-Loop (HITL)**: Explicitly add a flow to require the exporter to confirm the LLM-extracted APIs before compiling the `.xml` / `.pdf` packing list.
2. **Define the Hybrid Pipeline**: Clarify that Gemini outputs structured JSON (e.g., `{"ingredient": "Amoxicillin"}`) which is then passed to a traditional deterministic fast-lookup database, rather than asking Gemini to make the compliance decision directly.
3. **Async Processing UX**: Address the LLM latency by introducing UI loaders, background workers, or polling mechanisms in the P0 milestone.
4. **Data Security Pledge**: Automatically add a non-functional constraint stating that all document AI usage enforces strict training-data retention block policies.
